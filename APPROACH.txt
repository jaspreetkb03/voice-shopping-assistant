I built a lightweight, mobile-friendly single-page app using vanilla HTML/CSS/JavaScript so the project is easy to review and deploy within the assignment constraints. Voice input uses the browser Web Speech API (with language selection) to capture user commands. A compact parsing layer interprets common intents ("add", "remove", "find") and extracts quantities via regular expressions. Items are categorized using simple keyword maps so the UI groups grocery types with near-zero latency. User actions and history persist to localStorage, which enables smart suggestions: frequently purchased items and seasonal produce are surfaced dynamically. Substitutes are provided from a small lookup table. The app keeps all logic client-side to avoid deployment complexity and supports hosting on GitHub Pages or Firebase. Basic error handling accounts for unsupported browsers and microphone permission errors. The code is modular and commented to allow future improvements like connecting a product API for price filtering or using an NLP service for richer intent parsing.
